Master tmux windows for organizing tasks within a project.

## What Are Windows?

**Windows** are tmux's equivalent of tabs. Think of them as:
- Tabs in your terminal
- Different tasks within one project/session
- A collection of panes grouped together
- Named views for different workflows

**Hierarchy:**
```
Session → Windows → Panes
```

**Philosophy:** Windows organize related tasks within a project. If sessions are projects, windows are tasks.

## Creating Windows

### Basic Creation

**Create new window:**
```
C-Space c                   # Creates window, switches to it
```

**What happens:**
- New window starts in same directory (unless configured otherwise)
- Assigned next available number (0, 1, 2, ...)
- Becomes active window
- Starts with single pane

### Creating with Options

**From command line (outside tmux):**
```bash
tmux new-window -n "mywindow"                    # Named window
tmux new-window -n "server" -c ~/project         # With directory
tmux new-window -n "dev" "npm run dev"           # With command
```

**From inside tmux:**
```
C-Space :new-window -n "tests"                       # Named window
C-Space :new-window -c ~/other-project               # Different directory
```

## Navigating Windows

### Sequential Navigation

**Next/previous window:**
```
C-Space n                   # Next window (number increases)
C-Space p                   # Previous window (number decreases)
```

**How it works:**
- Cycles through window numbers: 1 → 2 → 3 → 1 (your config starts at 1)
- Skips missing numbers (if you close window 2: 1 → 3 → 1)

### Direct Navigation

**Jump to specific window:**
```
C-Space 1                   # Go to window 1 (your config starts at 1!)
C-Space 2                   # Go to window 2
C-Space 3                   # Go to window 3
...
C-Space 9                   # Go to window 9
```

**Note:** Your config has `base-index 1`, so windows start at **1, not 0**.

**For window numbers beyond 9:**
```
C-Space '                   # Prompts for window number
# Type: 15
# Press: Enter
```

### Smart Navigation

**Last window (toggle):**
```
C-Space l                   # Toggle between current and last window
# Like Alt+Tab in desktop
```

**Interactive window list:**
```
C-Space w                   # Opens window tree
# Use j/k or arrow keys
# Press Enter to select
```

**Your config shows window list:**
- All windows with their names
- Current window highlighted
- Preview of panes in each window
- Session information

## Window Management

### Renaming Windows

**Rename current window:**
```
C-Space ,                   # Prompts for new name
# Type name
# Press Enter
```

**Why rename?**
- **Default names** are the running program (bash, nvim, etc.)
- **Custom names** make navigation visual and clear
- **Examples:** "editor", "tests", "server", "logs", "git"

**From command:**
```
C-Space :rename-window newname
```

**Auto-naming:**
Your config has automatic window renaming. If you want to disable it:
```
C-Space :setw automatic-rename off
```

### Closing Windows

**Close current window:**
```
exit                    # In last pane, or
C-Space &                   # Prompts for confirmation (y/n)
```

**Kill without confirmation:**
```
C-Space :kill-window
```

**Kill specific window:**
```
C-Space :kill-window -t 2   # Kills window 2
```

**What happens when you close a window:**
- All panes in that window are closed
- Running processes are terminated
- Window number is freed
- You switch to next available window

### Moving Windows

**Swap windows:**
```
C-Space :swap-window -s 2 -t 5      # Swap window 2 with 5
C-Space :swap-window -t 1           # Swap current with window 1
```

**Move window to different number:**
```
C-Space :move-window -t 5           # Move current to position 5
```

**Move window to another session:**
```
C-Space :move-window -t othersession:    # Moves to "othersession"
```

**Useful for organizing:**
- Keep window 1 for editor (your config starts at 1!)
- Keep window 2 for tests
- Keep window 3 for server
- Reorder windows logically

### Linking Windows

**Link window from another session:**
```
C-Space :link-window -s othersession:2  # Links window from other session
# Now appears in both sessions!
```

**Use case:**
- Share monitoring window across sessions
- Common dashboard for multiple projects

## Window Layouts

### Manual Layout

**Typical project layout:**
```
Window 0: editor
  +------------------+
  |      nvim        |
  +------------------+

Window 1: tests
  +-------+----------+
  | Tests | Git      |
  |       | status   |
  +-------+----------+

Window 2: server
  +------------------+
  |   npm run dev    |
  +------------------+

Window 3: logs
  +------------------+----------+
  |                  |  System  |
  |   Application    |  logs    |
  |   logs           +----------+
  |                  |  Errors  |
  +------------------+----------+
```

### Window Numbering

**Default numbering:**
- Starts at 0
- Increments: 0, 1, 2, 3, ...

**Custom base index (start at 1 instead):**
```
# In tmux.conf:
set -g base-index 1

# Reload config:
C-Space r
```

**Your config likely starts at 0** (check your tmux.conf)

### Window Reordering

**Renumber windows sequentially:**
```
C-Space :move-window -r     # Renumbers all windows (removes gaps)
# If you have: 0, 2, 5, 8
# Becomes: 0, 1, 2, 3
```

**Auto-renumber on close:**
```
# In tmux.conf:
set -g renumber-windows on

# Now closing window 1 in [0,1,2,3] automatically makes [0,1,2]
```

## Common Window Workflows

### Workflow 1: Standard Development

**Setup:**
```
Session: myproject

Window 0: editor
  - Single pane
  - nvim

Window 1: terminal
  - Split horizontal
  - Top: general commands
  - Bottom: git status

Window 2: server
  - Single pane
  - npm run dev (or cargo run, go run, etc.)

Window 3: tests
  - Split vertical
  - Left: test watcher
  - Right: test output
```

**Navigate:**
```
C-Space 1                   # Editor
C-Space 1                   # Commands/Git
C-Space 2                   # Check server logs
C-Space 3                   # Check tests
```

**Why this works:**
- Clear separation of concerns
- Easy mental model (0=edit, 1=run, 2=serve, 3=test)
- Can maximize any window without affecting others

### Workflow 2: Frontend + Backend

**Setup:**
```
Window 0: editor (code)
  Split vertical: frontend code | backend code

Window 1: frontend-dev
  npm run dev (port 3000)

Window 2: backend-dev
  npm run server (port 8000)

Window 3: logs
  Split horizontal: frontend logs | backend logs

Window 4: database
  psql or mongo shell
```

**Quick switches:**
```
C-Space 1                   # Edit code
C-Space 1                   # Check frontend
C-Space 2                   # Check backend
C-Space 3                   # Debug logs
C-Space 4                   # Query database
```

### Workflow 3: Multi-Service Microservices

**Setup:**
```
Window 0: editor

Window 1: auth-service
Window 2: api-service
Window 3: worker-service
Window 4: queue-service

Window 5: monitoring
  Split into 4 panes showing all service logs

Window 6: database
```

**Name windows by service:**
```
C-Space ,                   # Rename each window
```

**Visual in status bar:**
```
[myproject] 0:editor 1:auth 2:api 3:worker 4:queue 5:monitor 6:db
```

### Workflow 4: Documentation + Implementation

**Setup:**
```
Window 0: docs
  - less or bat on documentation file
  - Or split: API docs | Implementation examples

Window 1: editor
  - Your code

Window 2: repl
  - Language REPL for testing
  - node, python, irb, etc.

Window 3: tests
  - Test runner
```

**Workflow:**
```
C-Space 1                   # Read docs
C-Space 1                   # Implement feature
C-Space 2                   # Test in REPL
C-Space 3                   # Run tests
C-Space l                   # Toggle between current and last
```

### Workflow 5: Git-Heavy Development

**Setup:**
```
Window 0: editor

Window 1: git
  Split 3-way:
  +--------+--------+
  |  diff  | status |
  +--------+--------+
  |    lazygit       |
  +------------------+

Window 2: tests
Window 3: server
```

**Or simpler with floating window:**
```
Window 0: editor
Window 1: tests
Window 2: server

# Use floating lazygit:
C-Space C-g                 # Opens lazygit in float
# No permanent window needed!
```

### Workflow 6: Comparison/Review

**Setup:**
```
Window 0: main-branch
  cd ~/project && git checkout main
  nvim

Window 1: feature-branch
  cd ~/project && git checkout feature-123
  nvim

Window 2: diff
  git diff main..feature-123
```

**Compare easily:**
```
C-Space 1                   # See main
C-Space 1                   # See feature
C-Space l                   # Toggle between them
C-Space 2                   # See diff
```

### Workflow 7: Learning New Tech

**Setup:**
```
Window 0: tutorial
  - less or browser (w3m, lynx)
  - Tutorial or docs

Window 1: practice
  - Editor for practice code

Window 2: repl
  - Interactive exploration

Window 3: reference
  - Man pages, examples, cheatsheets
```

### Workflow 8: Remote Server Management

**Setup:**
```
Session: production-server

Window 0: ssh
  ssh user@server

Window 1: logs
  ssh user@server "tail -f /var/log/app.log"

Window 2: monitoring
  Split 4-way:
  - htop
  - iostat
  - netstat
  - disk usage

Window 3: backup
  rsync commands, etc.
```

**Detach and reattach anytime!**

## Advanced Window Techniques

### Window Groups

**Create related windows:**
```bash
# Script: new-project-windows.sh
SESSION="myproject"

# Create session with first window
tmux new -s $SESSION -n editor -d

# Add more windows
tmux new-window -t $SESSION -n tests
tmux new-window -t $SESSION -n server
tmux new-window -t $SESSION -n logs

# Setup each window
tmux send-keys -t $SESSION:editor "nvim" C-m
tmux send-keys -t $SESSION:tests "npm test -- --watch" C-m
tmux send-keys -t $SESSION:server "npm run dev" C-m

# Attach to session
tmux attach -t $SESSION
```

### Window Monitoring

**Monitor for activity:**
```
C-Space :setw monitor-activity on
# Status bar shows when output happens in background window
```

**Monitor for silence:**
```
C-Space :setw monitor-silence 60    # Alert after 60 seconds of no output
# Useful for long-running builds
```

**Bell monitoring:**
```
C-Space :setw monitor-bell on       # Alert on terminal bell
```

**Visual indicator in status bar:**
- `#` - Activity detected
- `~` - Silence detected
- `!` - Bell detected

### Broadcast to Windows

**Send command to all windows:**
```bash
# From outside tmux:
tmux list-windows -t mysession -F "#{window_index}" | while read window; do
    tmux send-keys -t mysession:$window "git pull" C-m
done
```

**Useful for:**
- Updating all services
- Restarting all servers
- Running same command everywhere

### Window Layouts (Predefined)

**Save window layout:**
```
C-Space :list-windows       # Shows window info including layout string
# Copy the layout string (long hex string)
```

**Apply layout to window:**
```
C-Space :select-layout "layout-string-here"
```

**Use case:**
- Complex pane arrangements
- Reproducible setups

## Window vs Pane Decision

### When to Use Windows

**Good reasons:**
- **Different tasks** (editing, testing, serving)
- **Mental separation** (code vs logs vs database)
- **Full focus** (maximize one task without hiding others)
- **Independent workflows** (can close/restart without affecting others)

**Example:** Editor, tests, and server each deserve their own window

### When to Use Panes

**Good reasons:**
- **Related information** (code + test results side-by-side)
- **Visual comparison** (before vs after)
- **Monitoring while working** (editor + logs visible simultaneously)

**Example:** Code on left, tests on right

### Decision Matrix

| Scenario | Windows | Panes |
|----------|---------|-------|
| Editor for 3 microservices | ✓ | ✗ |
| Editor + test results | ✗ | ✓ |
| Dev server + worker + queue | ✓ | ✗ |
| Code + git status | Either | ✓ |
| Frontend + backend servers | ✓ | ✗ |
| Before/after comparison | ✗ | ✓ |
| Logs from 3 services | ✗ | ✓ |

### Hybrid Approach (Recommended)

```
Window 0: editor (single pane - focused coding)

Window 1: execution (split panes - related processes)
  +--------+--------+
  | Server | Worker |
  +--------+--------+

Window 2: monitoring (multiple panes - dashboards)
  +-------+-------+-------+
  | Logs  | CPU   | Net   |
  +-------+-------+-------+

Window 3: database (single pane - focused queries)
```

**Why this works:**
- Windows separate major concerns
- Panes group related information
- Balance between focus and context

## Troubleshooting

**"Window disappeared"**
```
C-Space w                   # List all windows
# Did you accidentally close it?
# Create new one: C-Space c
```

**"Lost track of which window is which"**
```
C-Space w                   # Shows tree with window names
# Or rename them all:
C-Space 1 C-Space , # Name: editor
C-Space 1 C-Space , # Name: tests
# etc.
```

**"Wrong window numbers (gaps)"**
```
C-Space :move-window -r     # Renumber sequentially
# Or enable auto-renumber in config
```

**"Window won't close"**
```
# Process running that ignores SIGHUP
# Force kill all panes:
C-Space :kill-window
# Or force kill window by number:
C-Space :kill-window -t 2
```

**"Accidentally closed important window"**
```
# No undo, sorry!
# Recreate it:
C-Space c
C-Space ,                   # Rename
# Re-run commands
```

**Prevention:** Use sessions for important work, detach instead of closing

**"Can't see window names in status bar"**
```
# Terminal too small, or
# Too many windows (names truncated)

# Check your tmux.conf window-status-format
# Or make terminal wider
```

**"Window numbering confusing"**
```
# Change base-index in tmux.conf:
set -g base-index 1     # Start at 1 instead of 0
C-Space r                   # Reload config
```

**"Want to switch faster than C-Space n/p"**
```
# Use direct numbers: C-Space 1, C-Space 2, C-Space 3
# Or SessionX: C-Space o
# Or window tree: C-Space w
```

## Best Practices

### 1. Name Your Windows

**Don't accept default names:**
```
# Bad:
bash, bash, bash, nvim, bash

# Good:
editor, tests, server, logs, db
```

### 2. Consistent Numbering

**Develop a standard layout:**
```
Window 0: Always editor
Window 1: Always tests
Window 2: Always server
Window 3: Always aux (git, docs, etc.)
```

**Muscle memory builds fast!**

### 3. Keep Window Count Reasonable

**Optimal:**
- 3-5 windows for most projects
- Easy to remember and navigate

**Too many:**
- 10+ windows gets confusing
- Use sessions instead (one per project)

### 4. Use Windows for Tasks, Sessions for Projects

**Good organization:**
```
Session: api-project
  Window 0: editor
  Window 1: tests
  Window 2: server

Session: frontend-project
  Window 0: editor
  Window 1: dev-server
  Window 2: storybook
```

### 5. Detach, Don't Close

**When switching projects:**
```
C-Space d                   # Detach (preserves all windows)
# Not: C-Space & to kill windows

# Come back later:
tmux attach -t myproject
# Everything exactly as you left it!
```

### 6. Monitor Important Windows

```
C-Space :setw -t 2 monitor-activity on     # Monitor window 2
# Status bar shows when output happens
```

### 7. Use Window Tree for Overview

```
C-Space w                   # See all sessions, windows, panes
# Great for "where am I?" moments
```

---

**Pro Tip:** Think of windows as "workspaces within a project". If you find yourself with 10+ windows, you probably need multiple sessions instead!
